This demo file contains some CSS examples.

CSS can be done in one of three ways

First, you can use a style attribute of an HTML element, e.g., this is an example using it.

Second, you can define a style as part of the head tag as I did above for the b tag. With that "b" tag defined this part will be in red as you can see.

Third, you can create a CSS file, e.g., demo.css, as I did in this example and link it. See the link tag defined above. It has three attributes: href, rel, and type. The href attribute has the CSS file name as its value.

There may be some things that are not obvious at this point. Let's see what each selector says in demo.css.

    body {font-size:12pt}
    h1,h2,h3 {color:purple}
    b {background-color:#b0c4de}
    tt {color:blue;font-size:12pt}
    #me {color:green}
    .pink {background-color:pink}

The body selector says the entire body should be in 12pt font size, which is easy to understand.

The second line defines selectors h1, h2, and h3 to be of color purple. You will notice that the first line of this document, i.e., A Quick Introduction is in purple as expected because it is in an h1 tag.

How come the second line (to HTML) with the h2 tag is in green instead of purple as is defined in demo.css? Well, it is because that line with the h2 tag is given an id with the value me and me is defined to be in green with the pound sign (#) in demo.css. We use the pound sign (#) to introduce a user-defined id selector as this example shows. As you can see, there is precedence order among these specifications. As you can see, we can also define one called a class selector using a dot (.) notation as is done with the pink selector. The difference between the ones defined by a pound sign and the ones defined by a dot is that the one defined by a pound sign is for a single unique element within a page, whereas the one defined by a dot is for a class of elements within a page.
